# Scenario

A user reported receiving a suspicious email claiming to be from Microsoft.

The email warned of a login attempt and instructed the user to reset their password.

Sender:
security-alert@micr0soft-support.com

The domain raised suspicion due to:
- Character substitution (0 instead of o)
- Use of a trusted brand name
- Urgent security language

Hypothesis:
The email is part of a phishing campaign designed to harvest credentials.
